# 🤖 בוט קריפטו - ניתוח טכני ומסחר

בוט קריפטו מקצועי בפייתון לניתוח טכני, חישוב אינדיקטורים ואיתותי מסחר.

## ✨ תכונות

- 🔌 חיבור ל-API של בורסות קריפטו (Binance, Bybit, OKX ועוד)
- 📊 ניתוח טכני עם אינדיקטורים: EMA, RSI, MACD
- 🎯 איתותי מסחר אוטומטיים (BUY/SELL/HOLD)
- 📈 גרפים מקצועיים עם הדגשת נקודות כניסה/יציאה
- ⏰ לולאת בדיקה רציפה או ניתוח חד-פעמי
- 🔒 תמיכה במסחר Spot (ניתן להרחיב ל-Futures)

## 🚀 התקנה

### 1. התקנת Python
ודא שיש לך Python 3.8 ומעלה מותקן:
```bash
python --version
```

### 2. התקנת ספריות
```bash
pip install -r requirements.txt
```

הספריות שיותקנו:
- `ccxt` - חיבור ל-API של בורסות
- `pandas` - ניהול וניתוח נתונים
- `ta` - אינדיקטורים טכניים
- `matplotlib` - ציור גרפים

### 3. הגדרת מפתחות API

ערוך את קובץ `config.py` והזן את מפתחות ה-API שלך:

```python
EXCHANGE_NAME = 'bybit'  # או 'binance', 'okx' וכו'
API_KEY = 'המפתח_שלך_כאן'
API_SECRET = 'הסוד_שלך_כאן'
```

⚠️ **חשוב**: 
- אל תשתף את מפתחות ה-API שלך עם אחרים
- השתמש רק ב-API Keys עם הרשאות קריאה (Read-only) לבדיקות ראשוניות
- הוסף את `config.py` ל-`.gitignore` (כבר מוגדר)

### 4. קבלת מפתחות API

**Bybit:**
1. היכנס לחשבון שלך ב-Bybit
2. לך ל-API Management
3. צור API Key חדש
4. העתק את ה-API Key וה-Secret Key

**Binance:**
1. היכנס לחשבון שלך ב-Binance
2. לך ל-API Management
3. צור API Key חדש
4. העתק את ה-API Key וה-Secret Key

## 📊 אינדיקטורים

הבוט משתמש באינדיקטורים הבאים:

### EMA (Exponential Moving Average)
- חלון: 50 תקופות (ניתן לשנות ב-`config.py`)
- שימוש: זיהוי מגמה

### RSI (Relative Strength Index)
- חלון: 14 תקופות
- שימוש: זיהוי מצבי על-קנייה/על-מכירה
- RSI < 30: על-מכירה (איתות קנייה)
- RSI > 70: על-קנייה (איתות מכירה)

### MACD (Moving Average Convergence Divergence)
- שימוש: זיהוי שינויי מגמה

## 🎯 אסטרטגיית מסחר

הבוט משתמש באסטרטגיה בסיסית:

**איתות קנייה (BUY):**
- RSI < 30 (על-מכירה)
- מחיר מעל EMA 50
- RSI יורד (מתקרב לנקודת קנייה)

**איתות מכירה (SELL):**
- RSI > 70 (על-קנייה) **או**
- מחיר מתחת ל-EMA 50

**אין איתות (HOLD):**
- כל שאר המקרים

⚠️ **הערה חשובה**: זו אסטרטגיה בסיסית לדוגמה. לפני מסחר אמיתי, מומלץ:
- לבדוק את האסטרטגיה על נתונים היסטוריים (Backtesting)
- להוסיף Stop-loss ו-Take-profit
- לבדוק את האסטרטגיה במצב Paper Trading
- להתחיל עם סכומים קטנים

## ⚙️ הגדרות

ניתן לשנות את ההגדרות בקובץ `config.py`:

```python
SYMBOL = 'BTC/USDT'  # זוג המטבעות
TIMEFRAME = '1h'  # מסגרת זמן: '1m', '5m', '15m', '1h', '4h', '1d'
LIMIT = 100  # מספר נתונים היסטוריים
EMA_WINDOW = 50  # חלון EMA
RSI_WINDOW = 14  # חלון RSI
RSI_OVERSOLD = 30  # סף על-מכירה
RSI_OVERBOUGHT = 70  # סף על-קנייה
CHECK_INTERVAL = 3600  # זמן המתנה בין בדיקות (שניות)
```
